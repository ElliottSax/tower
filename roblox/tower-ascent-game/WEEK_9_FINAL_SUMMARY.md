## üí° Design Philosophy Summary

### Progressive Learning Curve

**Easy (10%):**
- Introduce ONE hazard type (spikes)
- Obvious placement, easily avoidable
- Teaches "hazards exist and hurt"

**Medium (40%):**
- Multiple hazard types (quicksand, spikes, rotating)
- Integrated into section challenges
- Tests hazard avoidance + platforming

**Hard (60%):**
- Environmental hazards (ice, wind)
- Hazards affect core mechanics
- Requires adaptation and control mastery

**Expert (80%):**
- Lethal hazards (lava = instant death)
- Multi-hazard combinations
- Ultimate test of all skills

### Theme Consistency

**Grasslands ‚Üí** Safe, tutorial environment
**Desert ‚Üí** Environmental challenges (quicksand, heat)
**Snow ‚Üí** Control challenges (ice, wind)
**Volcano ‚Üí** Lethal environment (lava, ash, danger)

### Finale Emphasis

Each tier's final section uses **multi-hazard combinations**:
- Medium: Spikes + Quicksand
- Hard: Ice + Wind + Spikes
- Expert: **ALL 4 HAZARD TYPES**

**Attribute-Based System:**
```lua
:AddLava(position, size)
-- Creates part with HazardType = "Lava"
-- HazardService auto-detects and initializes
```

**No Per-Section Logic:**
- Hazards are data, not code
- HazardService handles all behavior
- Easy to add new hazard types
- Scales infinitely

**Theme Integration:**
- Hazards use theme-appropriate materials/colors
- Automatic particle/sound effects
- Consistent visual language
