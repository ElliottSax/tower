## Stat Balance Philosophy

### Power Scaling by Rarity

| Rarity | Base Power | Line Bonus | Combo Mult |
|--------|-----------|------------|------------|
| Common | 1.0-1.1x | 1.0-1.2x | 1.2-1.5x |
| Rare | 1.1-1.3x | 1.2-1.4x | 1.3-1.6x |
| Epic | 1.3-1.5x | 1.4-1.6x | 1.4-1.8x |
| Legendary | 1.6-1.8x | 1.7-1.9x | 1.7-2.0x |

**Design Goal**: Each tier is 30-50% more powerful than previous

### Ability Cooldowns

- Fast (25-30s): Common/Rare clear abilities
- Medium (35-45s): Rare/Epic clear abilities
- Slow (45-60s): Legendary abilities, resource generation

**Design Goal**: Legendary heroes = more power, similar cooldowns (feel impactful immediately)

### Ability Types Distribution

- **ClearRandomCells**: 4 heroes (Mage, Assassin, Necromancer, Archmage)
  - Progression: 8 → 12 → 18 → 25 cells
- **ClearRow**: 3 heroes (Knight, Paladin, Samurai)
  - Progression: 1 → 2 → 3 rows
- **ClearColumn**: 1 hero (Archer)
  - Unique mechanic
- **DoublePoints**: 2 heroes (Monk, Elementalist)
  - Duration: 10s → 15s
- **ExtraBlock**: 1 hero (Druid)
  - Resource generation
- **Shuffle**: 1 hero (Phoenix)
  - Board reset mechanic

1. **First 100 Gems** (Hour 2):
   - Player wants Archer or Monk

2. **First Rare Hero** (Hour 8):
   - Player wants Assassin/Paladin/Druid

## Hero Creation Checklist

For each hero in Unity:

1. **Create ScriptableObject**:
   - Assets → Create → Block Blast → Hero
   - Name: `Hero_<Name>` (e.g., `Hero_Knight`)

2. **Configure Identity**:
   - Set heroId (lowercase, underscores)
   - Set heroName (display name)
   - Write heroDescription (2-3 sentences)
   - Assign heroIcon sprite
   - Set rarity

3. **Configure Stats**:
   - Set basePower (refer to balance table)
   - Set lineBonus (refer to balance table)
   - Set comboMultiplier (refer to balance table)

4. **Configure Ability**:
   - Set abilityId (matches heroId pattern)
   - Set abilityName (display name)
   - Write abilityDescription (1 sentence)
   - Set abilityType (enum)
   - Set abilityPower (cells/rows affected)
   - Set abilityCooldown (seconds)
   - Set abilityDuration (if timed ability)

5. **Configure Progression**:
   - Set maxLevel (50 for all)
   - Set baseUpgradeCost (100 for all)

6. **Configure Unlock**:
   - Set unlockCost (0, 100, 250, 500, 1000)
   - Set unlockCurrency (Gems)
   - Check starterHero (Knight, Mage only)

7. **Assign to HeroSystem**:
   - Open main game scene
   - Select HeroSystem GameObject
   - Add hero to "All Heroes" array

8. **Test**:
   - Play mode
   - Use editor test: HeroSystem → "Test: Unlock Hero"
   - Verify stats scale correctly with levels

## Testing Hero Balance

### Power Testing
1. Play 10 games with each hero
2. Track average score
3. Ensure Legendary heroes score 50-80% higher than Common

### Cooldown Testing
1. Track ability usage per game
2. Target: 3-5 uses per 8-minute session
3. Adjust cooldowns if too frequent/rare

### Unlock Progression Testing
1. Simulate F2P progression (no IAP)
2. Track gem earning rate
3. Ensure 1-2 unlocks per day (engaged players)

### Monetization Testing
